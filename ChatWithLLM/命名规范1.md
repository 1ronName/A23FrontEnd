> Component name "Home" should always be multi-word.

这个报错来自 **Vue 的 ESLint 规则** `vue/multi-word-component-names`。意思是：

> **组件名应该至少有两个单词**，不要使用单个单词，比如 `"Home"`、`"Header"`、`"Footer"`。

---

### 原因

单词命名的组件容易与 HTML 原生标签冲突（比如 `<Home>` 看起来像 `<home>`）。多单词组件名可以避免歧义。

---

### 解决方法

1. **改组件文件名和 `name` 属性**：

```vue
<!-- Home.vue -->
<script setup lang="ts">
defineOptions({
  name: 'HomePage', // 改为多单词
})
</script>
```

或者老写法：

```vue
<script lang="ts">
export default {
  name: 'HomePage',
}
</script>
```

2. **也可以修改 ESLint 配置（不推荐）**
   在 `.eslintrc.js` 里：

```js
rules: {
  'vue/multi-word-component-names': 'off'
}
```

---

### 推荐做法

为了规范和防止和 HTML 标签冲突，**最好使用多单词命名**：

| 单词名 | 推荐多单词名   |
| ------ | -------------- |
| Home   | HomePage       |
| Upload | DocumentUpload |
| Result | ResultPage     |
